    Project History & Development

  • Introduce project

    • Project Inception

  • Dr. Tydings approached Technology with idea
  • Started February 2016
  • Dr. Tydings, who was our president at the time, approached the Technology Division with the project...Athens Tech had developed a similar system, which they referred to as the "panic" button.
  • She approached our Associate VP with the possibility of developing something similar, and he approached me "We can do this, right?" :)
  • I began development in February of 2016
  • First prototype presented to President Tydings
  • Basic functionality, location from computer, ability to send message.

    • Continued Development and Improvements

  • Input provided from Campus Safety and Security
  • Improvements Developed by Technology
  • Came up with initial prototype and presented to Dr. Tydings
  • Met with Campus Safety and Security for their input
  • Implemented ideas from these meetings, as well as our own improvements

    • Testing

    • Performed Summer 2016
    • Tests were performed at various locations across campus
  • Began "live" testing in the summer of 2016
  • Testing consisted of me scheduling tests with campus police,My laptop and I traveled to various location on campus, fired an alert, and timed responses

    • Testing

    Tip: Make sure you communicate with campus police prior to testing!!!

  • Tip: make sure you alert campus police that you are testing the system. While we communicated with the police chief about our first test, he didn't tell his officers, and needless to say their adrenaline was pumping!
  • By the way, I'm not saying our campus police are like Barney Fife...they're great.

    • Testing Results

    • "Live" tests with Campus Police - 2 minutes average response time.
  • Testing results - campus police were very responsive and responded with an average response time of 2 minutes for all areas tested (tested all buildings on campus)

    • Technical Stuff!

    Technologies/Tools Used

    Why these technologies?

    Why QT/C++?

    • Cross-platform: can compile for Windows, Apple, Linux
    • I wanted to learn C++

    Why Drupal/NodeJS/Socket.io?

    • Drupal: Familiar with Drupal development/PHP
    • Drupal: Great user experience for customer
    • NodeJS: Needed an asynchronous event runtime that could communicate with my application
    • NodeJS: Wanted to learn NodeJS
    • Socket.io: Made bidirectional communication infinitely more simple to write
  • Drupal is our CMS/development framework
  • familiarity with Drupal/PHP, Ability to create nice, easy to manage user experience for the web side of the project
  • NodeJS allows for asynchronous communication, which was needed for the dispatch element of the application
  • Socket.io made it easy to build the asynchronous bi-directional component of the application

    Desktop Application

    • Built with Qt/C++
    • Gets information from logged in user/Active Directory
    • Sends alert to web backend (Drupal) using REST call
  • Cool thing is that there is a REST module for Drupal that is reasonably easy to set up.

    • Web Backend

    • Built in Drupal
    • Acts as dispatch monitor
    • When alert is received, handles sending of messages to campus police
    • Once running, will alert the dispatcher audibly and visually with a popup message (NodeJS, Socket.io)
    • Emergency Backend

    Application Overview and Operation

    Basic Operation

    • Open the application by double-clicking Emergency Desktop Icon  
    • User clicks one of the presets, then clicks the "Send Alert" button.

    Campus Security Response

    • Campus security alerted via text message to security mobile phones.
    • Dispatcher notified via Web interface with audible warning and emergency details.
    • Security deployed to reported location.

    Lessons Learned

    Our Campus Police Rock!

  • I can't say enough about our campus police. They were very supportive and we could not have succeeded without them.

    • Communication is Key

    • Even with lots of communication, there was a little confusion
    • Excerpt of alert received: "This is just a notification to please make note that my office is now located in room HSC 1058 ( you had 1059 above)"

    Test, Test, and Test Again

    Limitations

    • Works well in situations where students can't see screen
    • Still difficult to use in a lab situation where instructor workstations are projecting to a screen
